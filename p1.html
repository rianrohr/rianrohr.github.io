CampusConnect UI Redesign

Creating intuitive solutions

  • Figma
  • Competitive Analysis
  • Wireframing
  • Prototyping
  • Usability Testing

Problem

The current student portal has a poorly structured design, creating an overwhelming user experience. Navigation is cluttered, making it difficult for students to access critical tools like class schedules, registration, and financial aid information quickly.

Challenge

Students need a user-friendly interface that enables them to easily view their schedules, register for courses, and make changes. The redesign must balance simplicity and functionality, ensuring an intuitive experience while retaining access to a wide range of tools.

What it looks like now...

The current CampusConnect interface lacks intuitive navigation and a clear hierarchy. Users often report difficulty locating basic resources, which leads to frustration and wasted time.

Key issues include:

  • Overwhelming navigation menu with no clear categories
  • Poor visual hierarchy that buries important features
  • Outdated design that doesn’t align with modern usability standards

How can we make the interface less challenging for users?

Insert survey research

Conducted a survey with 50 students, revealing that over 70% found the current interface confusing, with the top complaints being difficulty finding tools and slow navigation speed. Students prioritized quick access to schedules and registration tools.

Analyzed similar platforms from other universities, such as University of Michigan - Ann Arbor and University of Illinois at Urbana-Champaign, identifying effective navigation patterns, streamlined menu layouts, and modern design elements. These insights informed the design of CampusConnect’s updated interface.

Insert competitive analysis

Information Architecture

After initial research, I designed a user flow to create a logical and intuitive navigation system. The flow prioritized essential student tasks, such as accessing schedules, registering for courses, and viewing financial aid, while consolidating less-used features into secondary menus.

Interface Development

The development process began with low-fidelity sketches to map the layout of the redesigned interface. These sketches were iterated upon based on user feedback, eventually transitioning to medium- and high-fidelity prototypes created in Figma.

The redesigned landing page simplifies navigation by grouping core tools into clear categories, improving accessibility and reducing cognitive load for users.

The new menu bar features dropdown categories, ensuring students can quickly find the tools they need without navigating through cluttered menus.

The registration UI was overhauled to present available courses in a clean, easy-to-read format, with filters for departments, availability, and schedules.

Reflection and Conclusion

Through this project, I gained valuable insights into user-centered design and the importance of research in guiding solutions. Survey results helped accurately identify pain points, such as overwhelming navigation and poor accessibility, which informed targeted improvements in the redesign.

By conducting a competitive analysis, I identified design patterns that worked well in similar platforms, avoiding common pitfalls and focusing on usability. These findings translated into a streamlined interface with intuitive navigation, modern visuals, and a logical hierarchy of tools.

The final high-fidelity prototype demonstrated how students could seamlessly navigate through the redesigned CampusConnect, providing an efficient and user-friendly experience that met their needs.
